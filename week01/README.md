## Curly braces syntax

I noticed that in C#, unlike Javascript, the curly braces ( {} ) are written at the line of what we want to define. For instance, the following code block:

```csharp
if (condition)
{
    // statements here
}
```

that is really new to me so I hope to get used to it soon. I am thinking about practicing it often.

## C# is a strongly typed language

In contrast to Python and Javascript, C# is a **strongly type** language which means we have to define the type of each variable we declare. Though, it is not new to me since I have learned a bit of Swift
I am not yet used to it.

Additionally, once a type is assigned to a variable, it cannot be changed anymore. I still don't know if it is a good point or not yet. I will write about it later in the future 
if I figure it out during this course.

## What are all of these?

Well, there are a lot to digest in this first step of learning a new language. Once I open a C# file, I see a lot of keywords I don't know yet: void, static, using, etc... .
I see two sort of file in each folder: `Program.cs` and `[somethingSomething].csproj`. And other directories like `obj` and `bin`.

The first I learn from that is that in **C#** the main program file is called `Program` and not `main` like in Python. And then, that it looks like **Java** in the structure of the program.

## Habits to Change and Implement

While I was writing sample code in C#, I realized there are some habits from Python I had to change and others I had to implement:

1. *true* and not *True*
2. *Semicolons* at the end of each statement
3. Put *curly braces* at a new line
4. Clicking on the *Run* button at the top right to run the program instead of going to the **Debug menu**

I know, I know ... you may say that because I write code in Javascript, I should have gotten the habits of putting the semicolons already. What if I told you: not really.
Unfortunately, because Javascript allows developer to write code without semicolons, I have been used to write code without them. What a shame!
But I find it funny somehow that I am mandated to write semicolons, it's like a punishment for being lazy. I promise to be more disciplined from now on.

These makes me change how I have to write code regarding the language I am using. Due to the amount of time I have been programming in Python, it may take me some time for me to 
implement new habits. I will try to be patient anytime anyway.
